# L02 — Data Exploration with Pandas

## What I Did

Accessed and explored a structured dataset from GitHub directly in Google Colab. Built a data exploration pipeline using Pandas covering dataset loading, inspection, summary statistics, and initial analysis.

## What I Learned

Loading data directly from a GitHub repository removes the friction of manual file management and makes notebooks reproducible — anyone running the notebook gets the same dataset. Pandas' inspection methods (`head()`, `describe()`, `info()`, `value_counts()`) form a standard first-pass checklist that reveals shape, data types, missing values, and distribution in minutes.

## Challenges

Understanding which inspection methods to apply in what order — and what each result was actually telling me about the dataset's quality — required thinking about what questions I was trying to answer before writing any code.

## Technologies Used

Python · Pandas · NumPy · Google Colab · GitHub datasets

## How to Run

Open `L02_Francisco_Medina_ITAI2377.ipynb` in [Google Colab](https://colab.research.google.com/) and run all cells in order. The dataset is cloned automatically from GitHub.
